# MetaCrypt

**Шифрлауды қолдайтын, толығымен клиент жағында жұмыс істейтін ашық кодты метадеректер редакторы.**

## MetaCrypt не үшін керек?

Әр фотосурет көрінбейтін жүк алып жүреді: GPS координаттары, камераның
сериялық нөмірлері, уақыт белгілері, өңдеу тарихы. Метадеректерді көрсететін
құралдардың көбі **файлды серверге жүктеуді** талап етеді. MetaCrypt олай
істемейді — істей *алмайды*: сервер мүлдем жоқ. Барлығы браузерде File API,
типтелген массивтер және Web Crypto API арқылы жұмыс істейді, тіпті интернет
өшірулі болса да.

MetaCrypt мақсаты — фотографтар, әзірлеушілер, цифрлық криминалистика
мамандары мен архивистерге арналған эталондық ашық кодты метадеректер
редакторы болу. Ал суреттердің ішіне AES-256 арқылы шифрланған жазбаларды
жасыру мүмкіндігі — жалғыз сценарий емес, күшті қосымша функция.

## Мүмкіндіктер

- 🔍 **Қарау** — EXIF, XMP, IPTC, PNG мәтіндік чанктары, ICC, GPS, файл
  туралы мәліметтер, SHA-256 бақылау сомасы, RAW көріністері (JSON / XML / HEX / ағаш)
- ✏️ **Өңдеу** — өрістерді өзгерту, қосу, атын өзгерту және жою; өз XMP
  namespace-тері; EXIF мәндерін тип бойынша талдау (бөлшектер, GPS, күндер)
- 🧹 **Тазалау** — кез келген метадеректер тобын (немесе барлығын) бір
  басумен өшіру, бірде-бір пиксельді қайта сықпай
- 🔐 **Шифрлау** — Қорғалған қойма AES-256-GCM арқылы шифрланған жазбаны
  (логиндер, токендер, жазбалар, шексіз өз өрістері) суреттің XMP-іне енгізеді
- 📤 **Экспорт / импорт** — JSON, XML, TXT
- 📡 **Үйлесімділікті тексеру** — Telegram / WhatsApp / Drive / Discord…
  метадеректеріңізді (және шифрланған қойманы) сақтай ма?
- 🌍 **Көптілділік** — English, Русский, Қазақша; жүйе тілі автоматты түрде
  анықталады
- 🌓 **Қараңғы / жарық тақырып**, бейімделгіш дизайн, пернетақтамен басқару, ARIA
- 📴 **Орнатылатын PWA** — алғашқы кіруден кейін толығымен офлайн

## Жылдам бастау

Талаптар:

- Node.js 20+
- npm 10+

```bash
git clone <repository-url>
cd MetaCrypt
npm install
npm run dev
```

Содан кейін Vite көрсеткен URL-ді ашыңыз, әдетте `http://localhost:5173/`.

## Қалай қолдануға болады

1. **Редакторды** ашып, суретті жүктеу аймағына тастаңыз.
2. Табылған метадеректерді Overview, EXIF, XMP, IPTC, PNG және RAW беттерінен қараңыз.
3. Қолдау көрсетілетін өрістерді өзгертіңіз, қажетсіз метадеректерді тазалаңыз немесе JSON/XML/TXT ретінде экспорттаңыз.
4. Суретке шифрланған жазба енгізу керек болса, **Қойманы** ашыңыз.
5. Өзгертілген суретті жүктеп алыңыз. Пиксель деректері байт бойынша көшіріледі; тек метадеректер контейнерлері өзгереді.

Құпиялылықты тексеру үшін қолданба алғаш жүктелгеннен кейін DevTools → Network ашыңыз. Әдеттегі файл операциялары upload сұраныстарын жасамауы керек, себебі талдау, өңдеу және шифрлау браузердің ішінде орындалады.

## Қолдау көрсетілетін форматтар мен метадеректер

| Формат | EXIF | XMP | IPTC | PNG text | Тазалау |
|--------|------|-----|------|----------|---------|
| JPEG   | ✏️ оқу/жазу | ✏️ оқу/жазу | ✏️ оқу/жазу | — | ✅ |
| PNG    | 👁 оқу | ✏️ оқу/жазу | — | ✏️ оқу/жазу | ✅ |
| WebP / GIF / TIFF / HEIC / AVIF | 👁 оқу | 👁 оқу | 👁 оқу | — | — |

Ескертпелер:

- JPEG қысылған пиксельдерді өзгертпей, метадеректер сегменттерін ауыстыру арқылы өңделеді.
- PNG қазір мәтіндік/XMP чанктарын жазуды қолдайды. PNG ішіндегі `eXIf` бұл нұсқада тек оқылады.
- Кейбір мессенджерлер мен әлеуметтік платформалар метадеректерді әдейі өшіреді. Енгізілген қоймаға сенбес бұрын үйлесімділік бетін тексеріңіз.

## Шифрлау

Қорғалған қойма шифрланған жазбаны суреттің XMP пакетінің ішінде
`https://metacrypt.app/ns/1.0/` namespace-інде сақтайды:

```
құпиясөз ──PBKDF2-SHA256 (600 000 итерация, кездейсоқ 16 байт salt)──▶ AES-256 кілті
жазба JSON ──AES-256-GCM (кездейсоқ 96 бит IV, 128 бит auth tag)──▶ шифрмәтін
```

Конверт нұсқаланған: болашақ релиздер форматты өзгерте отырып, ескі
суреттердің шифрын аша алады. GCM деректерді аутентификациялайды: қате
құпиясөз немесе өзгертілген деректер бір ғана **жалпы** қате береді —
техникалық мәліметтер ашылмайды.

## Құпиялылық моделі

| Кепілдік | Қалай қамтамасыз етіледі |
|----------|--------------------------|
| Суреттер құрылғыдан шықпайды | желілік код жоқ; файлдар жадтағы `Uint8Array`-де өмір сүреді |
| Құпиясөздер құрылғыдан шықпайды | Web Crypto `deriveKey`, шығарылмайтын кілттер |
| Cookie жоқ, localStorage-те құпиялар жоқ | тек тақырып пен тіл сақталады |
| Шифры ашылған деректер логқа жазылмайды | деректер тек React күйінде, сіз жасырғанша ғана болады |
| Тексерілетіндік | MIT лицензиясы, ~60 шағын бастапқы файл, обфускациясыз |

## Қауіпсіздік ескертпелері

MetaCrypt — клиент жағындағы privacy құралы, бірақ толыққанды құпиясөз менеджерін немесе шифрланған backup жүйесін алмастырмайды.

- Шифрланған қойма жазбаның мазмұнын қорғайды, бірақ қойманың бар екені XMP ішінде көрінеді.
- Егер платформа метадеректерді өшірсе, қойма да солармен бірге өшеді.
- Ұмытылған құпиясөзді қалпына келтіру мүмкін емес.
- Браузер кеңейтімдері, бұзылған құрылғы, clipboard менеджерлері немесе экран жазбасы жұмыс кезінде деректерді көре алады.
- Маңызды файлдар үшін сенімді офлайн браузер профилін қолданыңыз және түпнұсқаларды бөлек сақтаңыз.

## Әзірлеу

```bash
npm install
npm run dev        # жергілікті dev-сервер
npm run build      # продакшен құрастыру (dist/) — GitHub Pages-ке дайын
npm run preview    # продакшен құрастыруды жергілікті қарау
```

Құрастыру салыстырмалы жолдарды (`base: './'`) және hash routing қолданады, сондықтан `dist/` кез келген статикалық хостингте және ішкі каталогта жұмыс істейді, соның ішінде GitHub Pages.

## Деплой

```bash
npm run build
```

`dist/` ішіндегіні кез келген статикалық хостингке жүктеуге болады.

GitHub Pages үшін жоба дайын:

- Vite `base: './'` қолданады.
- Роутинг hash URL арқылы жұмыс істейді, сондықтан бетті жаңарту маршруттарды бұзбайды.
- PWA файлдары `dist/` ішінде жасалады.

Қарапайым нұсқа:

1. Бастапқы кодты GitHub-қа жіберіңіз.
2. `npm run build` іске қосыңыз.
3. `dist/` қалтасын GitHub Pages, GitHub Actions немесе `gh-pages` бұтағы арқылы жариялаңыз.

## FAQ

**Менің суретім шынымен еш жерге жүктелмей ме?**
Иә. Жұмыс кезінде DevTools → Network ашыңыз: беттің алғашқы жүктелуінен
кейін — нөл сұраныс. Қолданба тіпті PWA ретінде офлайн жұмыс істейді.

**Суретте MetaCrypt қоймасы бар екенін біреу көре ала ма?**
Иә — конверт метадеректері (алгоритм, salt, IV, шифрмәтін) кез келген XMP
оқу құралына көрінеді, бұл әдейі жасалған шешім. Құпиясөзсіз мазмұнды оқу
есептеу тұрғысынан мүмкін емес. Есіңізде болсын: суреттерді қайта сығатын
платформалар (Telegram фото, WhatsApp) қойманы толығымен жояды.

**Құпиясөзімді ұмытып қалдым. Жазбаны қалпына келтіруге бола ма?**
Жоқ. Бэкдор да, қалпына келтіру де, сервер де жоқ. Мәні де осында.

**PNG EXIF өңдеу неге жұмыс істемейді?**
PNG EXIF-ті `eXIf` чанкында сақтайды; 1-нұсқа оны оқиды, бірақ әлі жазбайды.
