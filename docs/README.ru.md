# MetaCrypt

**Open-source редактор метаданных с поддержкой шифрования — полностью на стороне клиента.**

Просматривайте, редактируйте, очищайте, экспортируйте и *шифруйте* метаданные
изображений — на 100% в браузере. Без загрузок на сервер. Без бэкенда. Без трекинга.

## Зачем нужен MetaCrypt?

Каждая фотография несёт невидимый багаж: GPS-координаты, серийные номера
камеры, отметки времени, историю редактирования. Большинство инструментов для
просмотра метаданных требуют **загрузить файл на сервер**. MetaCrypt этого не
делает — он *не может*: сервера просто нет. Всё работает в браузере через
File API, типизированные массивы и Web Crypto API, и продолжает работать даже
с выдернутым сетевым кабелем.

Цель MetaCrypt — стать эталонным open-source редактором метаданных для
фотографов, разработчиков, специалистов по цифровой криминалистике и
архивистов. А возможность прятать внутри изображений записи, зашифрованные
AES-256, — сильная дополнительная функция, а не единственный сценарий.

## Возможности

- 🔍 **Просмотр** — EXIF, XMP, IPTC, текстовые чанки PNG, ICC, GPS,
  сведения о файле, контрольная сумма SHA-256, RAW-виды (JSON / XML / HEX / дерево)
- ✏️ **Редактирование** — изменение, добавление, переименование и удаление
  полей; пользовательские пространства имён XMP; типизированный разбор
  значений EXIF (дроби, GPS, даты)
- 🧹 **Очистка** — удаление любой группы метаданных (или всех сразу) в один
  клик, без пересжатия ни единого пикселя
- 🔐 **Шифрование** — Защищённое хранилище встраивает запись, зашифрованную
  AES-256-GCM (логины, токены, заметки, неограниченные свои поля), в XMP изображения
- 📤 **Экспорт / импорт** — JSON, XML, TXT
- 📡 **Проверка совместимости** — сохранит ли Telegram / WhatsApp / Drive /
  Discord… ваши метаданные (и зашифрованное хранилище)?
- 🌍 **Многоязычность** — English, Русский, Қазақша; язык системы
  определяется автоматически
- 🌓 **Тёмная / светлая тема**, адаптивность, навигация с клавиатуры, ARIA
- 📴 **Устанавливаемая PWA** — полностью офлайн после первого посещения

## Быстрый старт

Требования:

- Node.js 20+
- npm 10+

```bash
git clone <repository-url>
cd MetaCrypt
npm install
npm run dev
```

Затем откройте URL, который покажет Vite, обычно `http://localhost:5173/`.

## Как пользоваться

1. Откройте **Редактор** и перетащите изображение в область загрузки.
2. Посмотрите найденные метаданные во вкладках Overview, EXIF, XMP, IPTC, PNG и RAW.
3. Измените поддерживаемые поля, очистите лишние метаданные или экспортируйте их в JSON/XML/TXT.
4. Откройте **Хранилище**, если нужно встроить в изображение зашифрованную запись.
5. Скачайте изменённое изображение. Пиксели копируются байт-в-байт; меняются только контейнеры метаданных.

Для проверки приватности откройте DevTools → Network после первоначальной загрузки приложения. Обычные операции с файлами не должны создавать upload-запросов: парсинг, редактирование и шифрование выполняются в браузере.

## Поддерживаемые форматы и метаданные

| Формат | EXIF | XMP | IPTC | PNG text | Очистка |
|--------|------|-----|------|----------|---------|
| JPEG   | ✏️ чтение/запись | ✏️ чтение/запись | ✏️ чтение/запись | — | ✅ |
| PNG    | 👁 чтение | ✏️ чтение/запись | — | ✏️ чтение/запись | ✅ |
| WebP / GIF / TIFF / HEIC / AVIF | 👁 чтение | 👁 чтение | 👁 чтение | — | — |

Примечания:

- JPEG редактируется заменой сегментов метаданных без изменения сжатых пикселей.
- PNG сейчас поддерживает запись текстовых/XMP-чанков. `eXIf` в PNG в этой версии только читается.
- Некоторые мессенджеры и соцсети намеренно удаляют метаданные. Перед тем как полагаться на встроенное хранилище, проверьте вкладку совместимости.

## Шифрование

Защищённое хранилище сохраняет зашифрованную запись внутри XMP-пакета
изображения в пространстве имён `https://metacrypt.app/ns/1.0/`:

```
пароль ──PBKDF2-SHA256 (600 000 итераций, случайная 16-байтовая salt)──▶ ключ AES-256
JSON записи ──AES-256-GCM (случайный 96-битный IV, 128-битный auth tag)──▶ шифротекст
```

Конверт версионирован: будущие релизы смогут менять формат, продолжая
расшифровывать старые изображения. GCM аутентифицирует данные: неверный
пароль или подменённые данные дают одну **обобщённую** ошибку — никакие
технические детали не раскрываются.

## Модель приватности

| Гарантия | Как обеспечивается |
|----------|--------------------|
| Изображения не покидают устройство | сетевого кода нет; файлы живут в `Uint8Array` в памяти |
| Пароли не покидают устройство | Web Crypto `deriveKey` с неизвлекаемыми ключами |
| Без cookies, без секретов в localStorage | сохраняются только тема и язык |
| Расшифрованное не логируется | данные существуют только в состоянии React, пока вы их не скроете |
| Проверяемость | лицензия MIT, ~60 небольших файлов исходников, без обфускации |

## Заметки по безопасности

MetaCrypt — клиентский privacy-инструмент, но не замена полноценному менеджеру паролей или системе зашифрованных резервных копий.

- Зашифрованное хранилище скрывает содержимое записи, но сам факт наличия хранилища виден в XMP.
- Если платформа удаляет метаданные, хранилище удаляется вместе с ними.
- Забытый пароль восстановить нельзя.
- Расширения браузера, заражённое устройство, clipboard-менеджеры или запись экрана всё ещё могут увидеть данные во время работы.
- Для чувствительных файлов используйте доверенный офлайн-профиль браузера и храните оригиналы отдельно.

## Разработка

```bash
npm install
npm run dev        # локальный dev-сервер
npm run build      # продакшен-сборка (dist/) — готова для GitHub Pages
npm run preview    # локальный просмотр продакшен-сборки
```

Сборка использует относительные пути (`base: './'`) и hash-роутинг, поэтому
`dist/` работает с любого статического хостинга и из любого подкаталога —
включая GitHub Pages.

## Деплой

```bash
npm run build
```

Содержимое `dist/` можно загрузить на любой статический хостинг.

Для GitHub Pages проект уже настроен:

- Vite использует `base: './'`.
- Роутинг работает через hash URL, поэтому обновление страницы не ломает маршруты.
- PWA-файлы генерируются в `dist/`.

Простой вариант:

1. Загрузите исходники в GitHub.
2. Запустите `npm run build`.
3. Опубликуйте папку `dist/` через GitHub Pages, GitHub Actions или ветку `gh-pages`.

## Чеклист перед публикацией

## FAQ

**Моё изображение точно никуда не загружается?**
Да. Откройте DevTools → Network во время работы: после первоначальной
загрузки страницы — ноль запросов. Приложение даже работает офлайн как PWA.

**Можно ли увидеть, что в изображении есть хранилище MetaCrypt?**
Да — метаданные конверта (алгоритм, salt, IV, шифротекст) видны любому
XMP-ридеру, это осознанное решение. Без пароля содержимое вычислительно
нечитаемо. Помните: платформы, пересжимающие изображения (фото в Telegram,
WhatsApp), полностью уничтожают хранилище.

**Я забыл пароль. Можно восстановить запись?**
Нет. Нет ни бэкдора, ни восстановления, ни сервера. В этом и смысл.

**Почему редактирование PNG EXIF не работает?**
PNG хранит EXIF в чанке `eXIf`; версия 1 читает его, но пока не записывает.

## Участие

Issues и PR приветствуются. Держите изменения сфокусированными, типизированными и в существующем стиле. Каждый metadata-кодек должен безопасно проходить round-trip: нельзя записывать файл, который затем не получается распарсить. Перед отправкой запускайте `npm run build`.
